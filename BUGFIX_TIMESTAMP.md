## Проблема

Бот не находил сделки на активных рынках, всегда возвращая 0 сделок, даже когда API показывал наличие сделок.

### Симптомы

```
Found 0 trades in last 5 minutes for 0x37ec4c6b57a18b16eed1241f6155ee7ff45bc1697d7848f15ac33d406e38ed00
```

При этом API endpoint возвращал сделки:
```
https://data-api.polymarket.com/trades?market=0x37ec4c6b57a18b16eed1241f6155ee7ff45bc1697d7848f15ac33d406e38ed00
```

## Причина

В методе `get_trade_frequency()` была ошибка в обработке timestamp:

**Было (неправильно):**
```python
timestamp = trade.get("timestamp", 0)
# timestamp в миллисекундах, конвертируем в секунды
trade_time = timestamp / 1000  # ОШИБКА!
```

**Проблема:**
- API Polymarket возвращает timestamp уже в **секундах** (Unix timestamp)
- Деление на 1000 превращало timestamp в дату 1970 года
- Все сделки отфильтровывались как "слишком старые"

**Пример:**
- Реальный timestamp: `1776421518` (2026-04-17)
- После деления: `1776421.518` (1970-01-21)
- Текущее время: `1744970911` (2026-04-17)
- Результат: сделка "в прошлом" → отфильтрована

## Решение

**Стало (правильно):**
```python
timestamp = trade.get("timestamp", 0)
# timestamp уже в секундах (Unix timestamp)
trade_time = timestamp  # Используем как есть
```

## Технические детали

### Формат timestamp в Polymarket API

Согласно документации и реальным данным:
- **Формат:** Unix timestamp в секундах
- **Пример:** `1776421518` = 2026-04-17T10:25:18Z
- **Тип:** Integer (секунды с 1970-01-01)

### Правильная обработка

```python
import time

current_time = time.time()  # Текущее время в секундах
period_seconds = period_minutes * 60
cutoff_time = current_time - period_seconds

for trade in data:
    timestamp = trade.get("timestamp", 0)
    if timestamp >= cutoff_time:  # Сравниваем напрямую
        trade_count += 1
```
