## Задача

Добавить возможность фильтрации рынков по частоте совершения сделок с настраиваемым периодом от 1 до 10 минут.

## Реализованный функционал

### ✅ Основные компоненты

1. **База данных**
   - Добавлена таблица `trade_frequency_cache` для кэширования результатов
   - Методы `get_trade_frequency_cache()` и `save_trade_frequency_cache()`
   - Автоматическая миграция при запуске бота

2. **API интеграция**
   - Метод `PolymarketAPI.get_trade_frequency()` для получения данных о сделках
   - Использует официальный Polymarket Data API endpoint `/trades`
   - Подсчет сделок за указанный период времени
   - Кэширование результатов на 60 секунд

3. **Фильтрация**
   - Интеграция в `MonitoringService._check_filters()` для конкретных рынков
   - Интеграция в `AutoScanService._process_market()` для автосканирования
   - Асинхронная проверка частоты сделок
   - Поддержка периода от 1 до 10 минут

4. **Команды бота**
   - Обновлена `/set_filter` для поддержки `trade_frequency_min`
   - Обновлена `/filters` для отображения фильтра частоты
   - Обновлена `/help` с документацией по новому фильтру
   - Валидация параметров (период 1-10 минут, количество > 0)

5. **Тестирование**
   - Создан `test_trade_frequency.py` для проверки функционала
   - Тесты успешно пройдены
   - Проверена работа API, кэширования и фильтрации

## Технические детали

### Архитектура

```
User Command → Bot Handler → Database (filters)
                                ↓
MonitoringService/AutoScanService → PolymarketAPI.get_trade_frequency()
                                           ↓
                                    Polymarket Data API
                                           ↓
                                    Database (cache) ← 60s TTL
                                           ↓
                                    Filter Check → Notification
```

### API Endpoint

```
GET https://data-api.polymarket.com/trades?market={condition_id}&limit=1000
```

### Формат фильтра

```
filter_type: "trade_frequency_min"
filter_value: "{количество},{период_минут}"

Пример: "10,5" = минимум 10 сделок за 5 минут
```

### Кэширование

- **TTL:** 60 секунд
- **Ключ:** `condition_id` + `period_minutes`
- **Таблица:** `trade_frequency_cache`
- **Автоочистка:** устаревшие записи перезаписываются

## Использование

### Установка фильтра

```
/set_filter trade_frequency_min 10 5
```

### Примеры

```bash
# Очень активные рынки
/set_filter trade_frequency_min 20 5

# Умеренная активность
/set_filter trade_frequency_min 10 10

# Быстрые сделки
/set_filter trade_frequency_min 5 1

# Комбинация фильтров
/set_filter spread_min 2
/set_filter volume_min 100000
/set_filter trade_frequency_min 10 5
/auto_scan_start
```
