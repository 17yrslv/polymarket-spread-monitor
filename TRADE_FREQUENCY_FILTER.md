# Фильтр частоты сделок

## Описание

Новый фильтр `trade_frequency_min` позволяет отслеживать только те рынки, на которых совершается минимальное количество сделок за определенный период времени.

## Возможности

- ✅ Фильтрация по частоте сделок (от 1 до 10 минут)
- ✅ Работает в обоих режимах (мониторинг конкретных рынков + автосканирование)
- ✅ Кэширование результатов (60 секунд) для снижения нагрузки на API
- ✅ Использует официальный Polymarket Data API

## Использование

### Установка фильтра

```
/set_filter trade_frequency_min <количество_сделок> <период_минут>
```

**Примеры:**

```
/set_filter trade_frequency_min 10 5
```
Показывать только рынки с минимум 10 сделками за последние 5 минут

```
/set_filter trade_frequency_min 20 10
```
Показывать только рынки с минимум 20 сделками за последние 10 минут

```
/set_filter trade_frequency_min 5 1
```
Показывать только рынки с минимум 5 сделками за последнюю минуту

### Просмотр активных фильтров

```
/filters
```

Отобразит все активные фильтры, включая фильтр частоты сделок:
```
ID 1: Минимум 10 сделок за 5 мин
```

### Удаление фильтра

```
/remove_filter <id>
```

Или используйте кнопку "Удалить фильтр" в сообщении `/filters`

## Технические детали

### Архитектура

1. **PolymarketAPI.get_trade_frequency()** - получает данные о сделках из Polymarket Data API
2. **Database.trade_frequency_cache** - кэширует результаты на 60 секунд
3. **MonitoringService._check_filters()** - проверяет фильтр для конкретных рынков
4. **AutoScanService._process_market()** - проверяет фильтр при автосканировании

### API Endpoint

Используется официальный endpoint:
```
https://data-api.polymarket.com/trades?market={condition_id}&limit=1000
```

### Кэширование

- Результаты кэшируются на 60 секунд
- Снижает нагрузку на API при частых проверках
- Автоматическая очистка устаревших записей

### Ограничения

- Период: от 1 до 10 минут
- Минимальное количество сделок: от 1
- API лимит: 1000 последних сделок

## Примеры использования

### Поиск активных рынков

Найти рынки с высокой активностью (минимум 15 сделок за 5 минут):

```
/set_filter trade_frequency_min 15 5
/auto_scan_start
```

### Комбинирование фильтров

Найти активные рынки с хорошим спредом:

```
/set_filter spread_min 2
/set_filter volume_min 100000
/set_filter trade_frequency_min 10 5
/auto_scan_start
```

### Мониторинг конкретного рынка

Отслеживать только когда на рынке есть активность:

```
/set_market russia-ukraine-ceasefire-before-gta-vi-554
/set_filter trade_frequency_min 5 3
```

## Обновление базы данных

При первом запуске бота после обновления автоматически создается новая таблица:

```sql
CREATE TABLE IF NOT EXISTS trade_frequency_cache (
    condition_id TEXT PRIMARY KEY,
    last_check TIMESTAMP NOT NULL,
    trade_count INTEGER NOT NULL,
    period_minutes INTEGER NOT NULL
)
```

Миграция происходит автоматически, никаких дополнительных действий не требуется.
