# Автоматический мониторинг рынков Polymarket

## Что реализовано

Добавлена система автоматического сканирования рынков Polymarket с возможностью:
- Сканирования всех активных рынков или только выбранных категорий
- Фильтрации по спреду, объему и цене
- Автоматической отправки уведомлений при обнаружении подходящих рынков
- Дедупликации уведомлений (не отправлять повторно в течение N часов)

## Новые команды

### Управление автосканированием

**Основные команды:**
- `/auto_scan_start` - запустить автоматическое сканирование
- `/auto_scan_stop` - остановить автоматическое сканирование
- `/auto_scan_status` - показать статус и настройки

**Настройки:**
- `/auto_scan_mode <all|categories>` - режим сканирования
  - `all` - сканировать все активные рынки
  - `categories` - сканировать только выбранные категории
- `/auto_scan_interval <секунды>` - интервал проверки (по умолчанию 60 сек)
- `/auto_scan_dedup <часы>` - время дедупликации (по умолчанию 1 час)

### Управление категориями

- `/categories` - показать все доступные категории
- `/add_category <название>` - добавить категорию для сканирования
- `/remove_category <название>` - удалить категорию
- `/my_categories` - показать выбранные категории

## Примеры использования

### Пример 1: Сканирование всех рынков

```
/auto_scan_mode all
/set_filter spread_min 2.0
/set_filter volume_min 100000
/auto_scan_start
```

Бот будет сканировать все активные рынки каждую минуту и отправлять уведомления о рынках со спредом ≥2% и объемом ≥$100,000.

### Пример 2: Сканирование по категориям

```
/categories
/add_category Politics
/add_category Crypto
/auto_scan_mode categories
/set_filter spread_min 1.5
/auto_scan_dedup 2
/auto_scan_start
```

Бот будет сканировать только рынки категорий Politics и Crypto, отправлять уведомления о рынках со спредом ≥1.5%, не чаще раза в 2 часа для каждого рынка.

### Пример 3: Настройка интервала

```
/auto_scan_interval 300
/auto_scan_start
```

Бот будет проверять рынки каждые 5 минут (300 секунд).

## Технические детали

### База данных

Добавлены новые таблицы:
- `auto_scan_settings` - настройки автосканирования
- `scan_categories` - выбранные категории
- `notification_history` - история уведомлений для дедупликации

### Архитектура

- **AutoScanService** - сервис автоматического сканирования
  - Работает в фоновом режиме
  - Обрабатывает рынки батчами по 50 штук
  - Автоматически восстанавливается после перезапуска бота

- **PolymarketAPI** - расширен методами:
  - `get_markets_by_categories()` - получение рынков по категориям
  - `get_all_active_markets()` - получение всех активных рынков
  - `get_all_unique_categories()` - получение списка категорий

### Фильтрация

Используются те же фильтры, что и для мониторинга конкретных рынков:
- `spread_min` - минимальный спред (%)
- `volume_min` - минимальный объем ($)
- `yes_price_between` - диапазон цены Yes

### Дедупликация

Система отслеживает отправленные уведомления и не отправляет повторно уведомление о том же рынке в течение заданного времени (по умолчанию 1 час). История уведомлений автоматически очищается (старше 7 дней).

## Производительность

- Кэширование списка рынков (обновление раз в 5 минут)
- Батчинг запросов (обработка по 50 рынков параллельно)
- Обработка ошибок и retry логика
- Автоматическая очистка старых данных

## Ограничения

- Максимум 1000 рынков для сканирования (AUTO_SCAN_MAX_MARKETS)
- Минимальный интервал сканирования: 10 секунд
- Категории определяются полем `groupItemTitle` в API Polymarket

## Совместимость

Новая система автосканирования работает параллельно со старой системой мониторинга конкретных рынков. Вы можете использовать обе системы одновременно:
- Мониторинг конкретных рынков (до 5 штук) - `/set_market`
- Автосканирование всех/категорий - `/auto_scan_start`
