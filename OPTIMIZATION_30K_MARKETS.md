## 🎯 Цель

Увеличить количество сканируемых рынков с 10,000 до 30,000 при минимальной нагрузке на API и защите от rate limits.

---

## 📊 Что было изменено

### 1. Увеличен лимит страниц

**Файл:** `bot.py`, строка 48

```python
# БЫЛО:
AUTO_SCAN_MAX_PAGES = 10  # 10,000 рынков

# СТАЛО:
AUTO_SCAN_MAX_PAGES = 30  # 30,000 рынков
```

**Эффект:** Бот теперь сканирует до 30,000 рынков по умолчанию

---

### 2. Увеличен размер батча

**Файл:** `bot.py`, строка 47

```python
# БЫЛО:
AUTO_SCAN_BATCH_SIZE = 50

# СТАЛО:
AUTO_SCAN_BATCH_SIZE = 100
```

**Эффект:** 
- Обрабатывается по 100 рынков параллельно (было 50)
- Меньше итераций = меньше задержек
- Быстрее обработка

---

### 3. Увеличен TTL кэша

**Файл:** `bot.py`, метод `get_trade_frequency_cache()`

```python
# БЫЛО:
if datetime.now(timezone.utc) - last_check < timedelta(seconds=60):

# СТАЛО:
if datetime.now(timezone.utc) - last_check < timedelta(seconds=300):
```

**Эффект:**
- Кэш живет 5 минут вместо 1 минуты
- Меньше повторных запросов к API
- Экономия до 80% запросов при повторном сканировании

---

### 4. Оптимизированы задержки

**Файл:** `bot.py`

#### Задержка между страницами (строка 49):
```python
# БЫЛО:

# СТАЛО:
```

#### Задержка между батчами (строка ~1122):
```python

```

**Эффект:**
- Быстрее сканирование

### 5. Добавлена обработка rate limit (429)

**Файл:** `bot.py`, метод `fetch_with_retry()`

```python
elif resp.status == 429:

    wait_time = 10 * (attempt + 1)
```

**Эффект:**
- Автоматическая обработка rate limits
- Прогрессивное увеличение времени ожидания (10, 20, 30 секунд)

## ⏱️ Производительность

### До оптимизации (10,000 рынков):
- **Страниц:** 10
- **Батчей:** 200 (10,000 / 50)
- **Время батчей:** 200 × 0.5 = 100 сек
- **Время страниц:** 10 × 3 = 30 сек
- **Итого:** ~2-3 минуты

### После оптимизации (30,000 рынков):
- **Страниц:** 30
- **Батчей:** 300 (30,000 / 100)
- **Время батчей:** 300 × 0.3 = 90 сек
- **Время страниц:** 30 × 2 = 60 сек
- **Итого:** ~2.5-3 минуты (базовое сканирование)

### С учетом фильтра частоты сделок:
При использовании строгих фильтров (volume_min 500000):
- **Отсеивается:** ~95% рынков
- **Проверка частоты:** ~1,500 рынков
- **API запросов:** ~1,500
- **С кэшем (5 мин):** ~300-500 реальных запросов
- **Итого:** ~5-7 минут на полное сканирование

## 🛡️ Защита от rate limits

### 1. Увеличенный кэш (5 минут)
- Повторные сканирования используют кэш
- Экономия до 80% запросов

### 2. Обработка ошибки 429
- Автоматическая пауза при rate limit
- Прогрессивное увеличение времени ожидания
- Автоматическое восстановление

### 3. Безопасные задержки
- 0.3 сек между батчами
- 2 сек между страницами
- Не перегружает API

### 4. Оптимальный порядок фильтров
- Быстрые фильтры (spread, volume) идут первыми
- Медленный фильтр (trade_frequency) идет последним
- Минимум API запросов

## ⚠️ Важные замечания

### 1. Кэш обновляется каждые 5 минут
- Данные о частоте сделок могут быть устаревшими до 5 минут
- Это нормально для большинства случаев
- Если нужны более свежие данные, уменьшите TTL в коде

### 2. Rate limits
- API Polymarket может ограничивать запросы
- Используйте строгие фильтры для минимизации запросов
- Бот автоматически обрабатывает ошибку 429

### 3. Производительность
- Больше рынков = больше времени
- Используйте фильтры для оптимизации
- Мониторьте логи

### 4. Интервал сканирования
- По умолчанию: 60 секунд между сканированиями
- Можно увеличить для снижения нагрузки:
  ```
  /auto_scan_interval 120
  ```
