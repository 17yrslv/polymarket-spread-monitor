# 📊 Polymarket Spread Monitor Bot

Асинхронный Telegram бот для мониторинга спредов на рынках Polymarket в режиме реального времени.

## 🎯 Основные возможности

- ✅ Мониторинг до 5 рынков одновременно
- ✅ Обновление данных каждые 10 секунд
- ✅ Расчет bid-ask спреда из CLOB данных
- ✅ Гибкая система фильтров (минимальный спред, объем, диапазон цен)
- ✅ Автоматическое сохранение настроек в SQLite
- ✅ Работа через прокси для обхода блокировок
- ✅ Подробное логирование всех операций
- ✅ Retry логика для надежности API запросов

## 📋 Требования

- Python 3.8 или выше
- Telegram аккаунт
- Интернет соединение

## 🚀 Установка

### Шаг 1: Установка Python

Убедитесь, что у вас установлен Python 3.8+. Проверить можно командой:

```bash
python --version
```

Если Python не установлен, скачайте его с [python.org](https://www.python.org/downloads/)

### Шаг 2: Установка зависимостей

Откройте командную строку в папке проекта и выполните:

```bash
pip install -r requirements.txt
```

Это установит все необходимые библиотеки:
- aiogram 3.7.0 - фреймворк для Telegram ботов
- aiohttp 3.9.5 - асинхронный HTTP клиент
- aiosqlite 0.20.0 - асинхронная работа с SQLite
- loguru 0.7.2 - продвинутое логирование
- python-dotenv 1.0.1 - управление переменными окружения
- aiohttp-socks 0.8.4 - поддержка прокси

### Шаг 3: Запуск бота

#### Вариант 1: Через .bat файл (рекомендуется для Windows)

Просто дважды кликните на файл `start_bot.bat`

#### Вариант 2: Через командную строку

```bash
python bot.py
```

**Пример:**

/set_market russia-ukraine-ceasefire-before-gta-vi-554
```

**Ответ бота:**
```

### /set_filter <type> <value>
Установить фильтр для уведомлений. Уведомления будут отправляться только если данные соответствуют всем установленным фильтрам.

#### Типы фильтров:

**1. spread_min** - минимальный спред в процентах

Отправлять уведомления только если спред >= указанного значения.

```
/set_filter spread_min 1.5
```

**2. volume_min** - минимальный объем торгов в долларах

Отправлять уведомления только если объем >= указанного значения.

```
/set_filter volume_min 500000
```

**3. yes_price_between** - диапазон цены Yes контракта

Отправлять уведомления только если цена Yes находится в указанном диапазоне.

```
/set_filter yes_price_between 0.4 0.6
```

## 📨 Формат уведомлений

Каждые 10 секунд бот отправляет обновления по каждому рынку (если данные проходят фильтры):

```
🏛️ Рынок: Russia-Ukraine Ceasefire before GTA VI?
📊 Спред: 1.85% (Bid-Ask)
Yes: 0.540 | No: 0.460
Объём: $1,518,922
Ликвидность: $61,653
Последнее обновление: 16:00:15
```

**Расшифровка полей:**
- **Рынок** - название рынка на Polymarket
- **Спред** - разница между ценой покупки (bid) и продажи (ask) в процентах
- **Yes/No** - текущие цены контрактов Yes и No
- **Объём** - общий объем торгов за все время
- **Ликвидность** - доступная ликвидность в order book
- **Последнее обновление** - время отправки уведомления

## 🔍 Как найти slug рынка на Polymarket

### Способ 1: Использовать команду /search (рекомендуется)

Это самый простой способ! Просто введите ключевые слова:

```
/search Trump
/search Ukraine
/search Bitcoin
```

Бот найдет все подходящие рынки и покажет их slug.

### Способ 2: Скопировать из URL на сайте

1. Откройте [polymarket.com](https://polymarket.com)
2. Найдите интересующий вас рынок
3. Кликните на него, чтобы открыть страницу рынка
4. Посмотрите на URL в адресной строке браузера

**Пример URL:**
```
https://polymarket.com/event/russia-ukraine-ceasefire-before-gta-vi-554
```

**Slug (последняя часть после `/event/`):**
```
russia-ukraine-ceasefire-before-gta-vi-554
```

Скопируйте этот slug и используйте в команде `/set_market`

## 🛠️ Troubleshooting

### Бот не запускается

**Проблема:** При запуске появляется ошибка `ModuleNotFoundError`

**Решение:** Установите зависимости:
```bash
pip install -r requirements.txt
```

---

**Проблема:** Ошибка `python: command not found`

**Решение:** Python не установлен или не добавлен в PATH. Установите Python с [python.org](https://www.python.org/downloads/) и убедитесь, что отметили галочку "Add Python to PATH" при установке.

---

### Ошибки подключения к Telegram

**Проблема:** `Connection timeout` или `Cannot connect to Telegram`

**Решение:** Проверьте:
1. Интернет соединение
2. Прокси настройки в bot.py (строки 28-29)
3. Правильность токена бота (строка 28)

---

### Проблемы с прокси

**Проблема:** `ProxyConnectionError` или `Proxy authentication failed`

**Решение:** 
1. Проверьте правильность прокси URL в bot.py (строка 29)
2. Убедитесь, что прокси сервер доступен
3. Проверьте логин и пароль прокси

---

### Рынок не найден

**Проблема:** Бот отвечает `❌ Рынок с slug '...' не найден`

**Решение:**
1. Проверьте правильность slug (скопируйте из URL на polymarket.com)
2. Убедитесь, что рынок активен и не закрыт
3. Попробуйте найти рынок через поиск на Polymarket

---

### Нет уведомлений

**Проблема:** Бот запущен, но уведомления не приходят

**Решение:**
1. Проверьте статус: `/status` - мониторинг должен быть активен
2. Проверьте фильтры: `/filters` - возможно, данные не проходят фильтры
3. Проверьте список рынков: `/markets` - убедитесь, что рынки добавлены
4. Попробуйте удалить фильтры и добавить рынок заново

## 🏗️ Технические детали

### Архитектура

Бот построен на асинхронной архитектуре с использованием:
- **aiogram 3.x** - современный фреймворк для Telegram ботов
- **asyncio** - асинхронное выполнение задач
- **aiohttp** - асинхронные HTTP запросы к Polymarket API
- **aiosqlite** - асинхронная работа с базой данных

### Компоненты

**1. PolymarketAPI** - клиент для работы с Polymarket API
- Получение списка рынков
- Получение данных конкретного рынка
- Расчет bid-ask спреда
- Retry логика (3 попытки с экспоненциальной задержкой)

**2. Database** - управление SQLite базой данных
- Таблица users - пользователи бота
- Таблица markets - отслеживаемые рынки
- Таблица filters - фильтры пользователя
- Таблица monitoring_state - состояние мониторинга

**3. MonitoringService** - фоновый мониторинг
- Asyncio task для каждого пользователя
- Проверка рынков каждые 10 секунд
- Применение фильтров
- Отправка уведомлений

**4. Command Handlers** - обработчики команд
- Middleware для проверки User ID
- Обработка всех команд бота
- Валидация входных данных

### База данных

Бот использует SQLite для хранения:
- Список отслеживаемых рынков
- Настройки фильтров
- Состояние мониторинга

База данных создается автоматически при первом запуске в файле `bot.db`

### Логирование

Все операции логируются в:
- **Консоль** - уровень INFO (цветной вывод)
- **bot.log** - уровень DEBUG (подробное логирование)

Лог файл автоматически ротируется при достижении 10 MB и хранится 7 дней.

### Безопасность

- Доступ к боту ограничен одним User ID (6728174404)
- Все остальные пользователи получают "Access denied"
- Прокси для обхода блокировок Telegram API

### API Endpoints

Бот использует официальные Polymarket API:
- **Gamma API** - `https://gamma-api.polymarket.com/markets`
  - Получение списка всех рынков
  - Получение данных конкретного рынка
  - Bid/Ask цены, объем, ликвидность

### Расчет спреда

Спред рассчитывается по формуле:
```python
spread = ((bestAsk - bestBid) / bestBid) * 100
```

Где:
- `bestBid` - лучшая цена покупки
- `bestAsk` - лучшая цена продажи
